前言

AngularJS是一款来自Google的前端JS框架，起源于2009年的Google Feedback项目。Misko Hevery为AngularJS之父。
这款框架的核心特性有MVC、模块化、自动化双向数据绑定、语义化标签、依赖注入等。
AngularJS框架本身是通过TDD（测试驱动）的方式来开发的，从这个角度来看，AngularJS是敏捷开发的一次成功实践。
AngularJS让单元测试和集成测试成为可能。

简介

一些概念

客户端模板

在AngularJS中，模板和数据都会发送到浏览器中，然后在客户端进行装配。

MVC

MVC背后的核心理念是：将管理数据的代码(model)、应用逻辑代码(controller)、以及向用户展示数据的代码(view)清晰地分离开。
在AngularJS应用中，视图就是Document Object Model(DOM，文档对象模型)，控制器就是JavaScript类(function)，而模型数据则被存储在对象的属性中。

数据绑定

仅仅声明UI中的某个部分需要映射到某个JavaScript属性，然后让他们自己去同步。这种编程风格叫数据绑定。

依赖注入

依赖注入是一种软件设计模式，用来处理代码的依赖问题。
AngularJS的依赖注入只是简单的获取它所需要的东西，而不需要创建那些他们所依赖的东西。这种风格遵循了一种叫做迪米特法则（核心含义是一个类要尽量少地知道其他类的相关知识）的设计模式，也叫做最少知识原则。那么用这法则就意味着，它不应该去操心任何其他东西，例如$scope是如何创建的，以及到哪里找这个对象等。

指令

Angular在模板中定义的一些新属性，不属于HTML规范，Angular利用这些属性来实现各种功能，我们把这些叫做HTML扩展指令。
Angular内置了很多指令，我们也可以自定义指令。

AngularJS应用骨架

调用Angular

为了使用Angular，所有应用都必须首先做两件事情：
1)加载Angular.js库
2)使用ng-app指令告诉Angular应该管理DOM中的哪一个部分

Model View Controller

1)用来容纳数据的模型，模型代表当前的状态
2)用来展示数据的一些视图
3)用来管理模型和视图之间关系的一些控制器
对于大多数的应用来说，你都需要创建模型对象类来容纳你的数据。
定义控制器的正确方式是把它定义成模块的一部分，控制器可以为应用里面相关的部分提供命名空间机制。

模板和数据绑定

Angular应用中的模板只是一些HTML片段而已。
使用Angular构建应用的时候，可以将应用中的模板和数据分离开来。这样就可以把这些模板缓存起来。这样可以提升应用的性能。

显示文本

双花括号插值语法阅读起来比较自然，输入的内容也更少。但是在Angular使用数据替换这些花括号之前，其未被渲染好的模板可能被用户看到。此时，可以使用ng-bind指令。
在大多数模板中，依然可以使用双花括号，但是对于index.html页面的数据绑定操作，建议使用ng-bind。

浅谈非入侵式JavaScript

ng-click
这些指令和原来的事件处理器有以下不同之处：
1.在所有的浏览器中具有相同的行为，Angular将会帮你屏蔽差异性
2.不会在全局命名空间中进行操作。你所指定的表达式只能访问元素控制器作用域范围内的函数和数据

事件处理器会绑定数据结构和行为，但Angular并不会产生什么副作用，我们有一种简单的测试方法可以验证：我们能否为应用逻辑编写一个单元测试而不需要DOM结构存在。

迭代

ng-repeat $index $first $last $middle

显示和隐藏

ng-show ng-hide

CSS类和样式

可以这样设置类class=”munu-disabled-“,也可以这样设置内联样式，但最好使用ng-class或ng-style指令。这两个指令都可以接受一个表达式，表达式执行的结果可能是以下取值之一：
1.表示CSS类名的字符串，以空格分隔
2.CSS类名数组
3.CSS类名到布尔值的映射

反思src和href

ng-src ng-href

表达式

ng支持的表达式
1.简单的数学运算：+ - * / %
2.进行比较运算：== != > < >= <=
3.执行布尔逻辑: && || !
4.位运算：\^ & |
5.调用控制器中$scope对象上暴露的函数，引用数组和对象符号：[] {} .
不支持循环结构（for、while等）、流程控制操作符（if-else，throw）以及修改数据的操作符（++、–）

控制器

在应用中控制器有三个职责：
1.为应用中的模型设置初始状态
2.通过$scope对象把数据模型和函数暴露给视图（UI模板）
3.监视模型其余部分的变化，并采取相应的动作。
从概念上讲，控制器存在的目的是：当用户与视图进行交互时，它负责提供代码或逻辑以便执行用户的想法。

利用$scope暴露模型数据

可以显示地创建$scope属性，也可以间接地通过模板自身创建数据模型：
1.表达式：在表达式设置属性值和设置控制器中$scope的属性值是一样的

  1. 在表单输入项上使用ng-model

    使用$scope监控数据模型的变化

    $watch(watchFn,watchAction,deepWatch)

    使用Module（模块）组织依赖关系

    服务都是单例的对象，他们用来执行必要的任务，支撑应用的功能。
    在大多数的应用中，创建供所有代码使用的单个模块，并把所有依赖的东西都放入这个模块中，这样就能工作得很好。但是，如果你打算使用第三方包中所提供的服务或指令，他们一般会带有自己的模块。由于你的应用依赖这些模块，所有需要在应用模块中定义依赖关系才能引用它们。

    使用过滤器格式化数据

    可以自己编写过滤器：

    var homeModule = angular.module(‘homeModule’,[]);
    homeModule.filter(‘titleCase’,function(){

    var titleCaseFilter = function(input){
    var words = input.split(' ');
    for( var i = 0; i < words.length; i++){
        words[i] = words[i].charAt(0).toUpperCase() + words[i].slice(1);
    }
    return words.join(' ');
};
return titleCaseFilter;

    });

    根据URL导航的方式叫做深度链接
    /view/:id 在id前面加上一个冒号，从而指定了一个参数化的URL组件
    从长远来看，让服务来代理与服务器之间的交互的工作对我们有好处，这个服务可以被多个控制器共享。
